당신은 이 Replit 프로젝트(Streamlit 기반 “AI 연간 사업계획 통합 에이전트”)의 유지보수 개발자다.
현재 “분석 완료 후 결과가 전부 0/빈값”이 되는 치명적인 문제를 해결하라.

0) 문제 요약(재현 증거)

앱에서 PDF 2~10개 업로드 후 “AI 분석 시작”을 눌러도 결과가 채워지지 않으며, 화면상 대부분이 0/빈칸이다.

디버그 정보에서 각 업로드 PDF에 대해 다음이 반복된다.

extracted_len: 0

요약 텍스트 길이: 0자

즉, PDF에서 텍스트가 한 글자도 추출되지 않아 요약/분석 입력이 빈 문자열이 된다.

Shell에서도 *.pdf가 워크스페이스에 남지 않는 것이 정상이며(업로드 파일은 메모리로 처리), Shell로 원본 PDF 텍스트를 직접 보는 방식으로는 확인이 어렵다. 앱 내부 디버그 출력이 정답이다.

1) 원인 분석(추정이 아니라 코드 기반으로 확인)

utils.py의 PDF 텍스트 추출이 사실상 PyPDF2 한 가지 방식에 의존한다.

PDF가 스캔본(이미지 기반) 이거나, PyPDF2가 해당 PDF 구조의 텍스트를 못 읽는 경우 extract_text()가 빈 문자열을 반환한다.

이 상태에서 extract_file_summaries()는 입력 텍스트가 없어 요약을 생성하지 못하거나(빈값), summaries_to_compact_text()가 빈 문자열이 되어 최종 분석이 “빈 입력”으로 진행된다.

따라서 핵심 해결은 PDF 텍스트 추출 폴백 강화 + 실패 시 안내/진단 강화이다.

2) 목표(완료 기준)
A. 텍스트 PDF 대응

텍스트 기반 PDF라면 extracted_len이 0이 아니라야 한다.

요약 텍스트 길이가 0이 아닌 값이 되며, PART1~PART4 결과가 채워져야 한다.

B. 스캔(이미지) PDF 대응

텍스트가 추출되지 않으면 디버그 정보에 “스캔/이미지 PDF 가능, OCR 필요” 경고가 명확히 보여야 한다.

사용자가 왜 0이 나오는지 즉시 이해할 수 있어야 한다.

C. 안정성

어떤 PDF가 와도 앱이 죽지 않고(예외 처리), 문제를 진단/안내하며 다음 행동을 유도해야 한다.

4) utils.py 수정 ①: PDF 추출 폴백 강화(필수)
(A) 교체 대상 함수

utils.py에서 아래 함수 전체를 찾아 통째로 교체한다.

함수명: _extract_text_from_pdf_bytes(pdf_bytes: bytes) -> str

(B) 교체 코드(그대로 사용)
def _extract_text_from_pdf_bytes(pdf_bytes: bytes) -> str:
    """
    PDF 텍스트 추출 폴백:
    1) PyPDF2
    2) pdfplumber
    3) PyMuPDF(fitz)
    - 텍스트 기반 PDF는 최대한 추출
    - 스캔/이미지 PDF는 여기서 텍스트가 거의 나오지 않으며, 이후 단계에서 OCR 안내
    """
    if not pdf_bytes:
        return ""

    def _clean(s: str) -> str:
        return (s or "").replace("\x00", "").strip()

    def _good_enough(s: str) -> bool:
        # 공백 제거 기준 30자 이상이면 "추출 성공"으로 간주
        return len(re.sub(r"\s+", "", s or "")) >= 30

    # 1) PyPDF2
    try:
        from PyPDF2 import PdfReader  # type: ignore
        reader = PdfReader(io.BytesIO(pdf_bytes))
        parts = []
        for page in reader.pages:
            parts.append(page.extract_text() or "")
        txt = _clean("\n".join(parts))
        if _good_enough(txt):
            return txt
    except Exception:
        pass

    # 2) pdfplumber
    try:
        import pdfplumber  # type: ignore
        parts = []
        with pdfplumber.open(io.BytesIO(pdf_bytes)) as pdf:
            for p in pdf.pages:
                parts.append(p.extract_text() or "")
        txt = _clean("\n".join(parts))
        if _good_enough(txt):
            return txt
    except Exception:
        pass

    # 3) PyMuPDF (fitz)
    try:
        import fitz  # type: ignore
        doc = fitz.open(stream=pdf_bytes, filetype="pdf")
        parts = []
        for page in doc:
            parts.append(page.get_text("text") or "")
        txt = _clean("\n".join(parts))
        if _good_enough(txt):
            return txt
    except Exception:
        pass

    return ""

5) utils.py 수정 ②: 진단 정보(diagnostic) 추가(필수)
(A) 수정 대상

read_uploaded_file(uploaded_file) 함수

(B) 요구사항

반환 dict에 diagnostic 키를 추가하여 다음 정보를 포함한다.

pdf_text_extracted: bool

note: str (추출 실패 시 OCR 필요 안내)

가능하면 ext, mime, bytes, extracted_len

(C) 적용 방법

read_uploaded_file()의 마지막 return을 아래 패턴으로 변경한다.

    diagnostic = {
        "ext": ext,
        "mime": file_type,
        "bytes": size,
        "extracted_len": len(text or ""),
        "pdf_text_extracted": bool((text or "").strip()),
        "note": ""
    }

    if ("pdf" in file_type or ext == ".pdf") and not (text or "").strip():
        diagnostic["note"] = "PDF에서 텍스트 추출 실패(스캔/이미지 PDF 가능). OCR이 필요할 수 있습니다."

    return {"name": name, "size": size, "type": file_type, "text": text, "diagnostic": diagnostic}
