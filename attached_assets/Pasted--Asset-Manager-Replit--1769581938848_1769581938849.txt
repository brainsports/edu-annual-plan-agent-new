안녕하세요. 지금 Asset-Manager(Replit) 앱에서 “분류(구조화)는 정상인데, 지침 기반 재작성(특히 표/만족도)이 실패”하는 문제가 반복되어 수정이 필요합니다. 

제가 파악한 문제와 해결 방향은 아래와 같습니다. 
혹시 더 좋은 대안(더 단순하고 안정적인 구조, 더 높은 품질, 비용/속도 최적화)이 있다면 그 방식으로 진행해 주세요. 
만약 더 나은 대안이 없다면, 제가 제안한 방식대로 그대로 작업해 주세요.

────────────────────────────────────────
1) 현재 문제(증상)
────────────────────────────────────────
A. 본문 텍스트 영역은 성공
- PART1의 이용아동 욕구/지역 특성/사업목적/사업목표 등 “텍스트 영역”은 글자수 규칙을 잘 통과합니다.

B. 표 안의 셀 텍스트가 글자수 실패(주로 300자 조건)
- 환류계획 테이블 등 “표 셀” 안의 내용은 ‘공백 제외 최소 300자’ 같은 조건에서 자주 실패합니다.
- 불릿 개수는 맞추기도 하지만, 셀 내용이 짧아서 규칙이 깨집니다.

C. 만족도(주관식) 분석/제언이 글자수 실패(각 500자)
- “주관식 문항 요약분석(500자 이상)”, “종합 분석 및 제언(500자 이상)”이 최소 글자수 조건을 만족하지 못합니다.

D. 미리보기/검증 UI의 불필요 화면(그림1,2)은 삭제(또는 기본 숨김) 필요
- 사용자가 작성 결과 폼을 바로 보고 싶어 하는데, 중간 검증/로그 화면이 미리보기에서 방해됩니다.

────────────────────────────────────────
2) 원인에 대한 제 추정
────────────────────────────────────────
- guidelines(JSON)에서 “표 컬럼”은 max만 있고 min이 없거나, min 강제가 코드에 반영되지 않은 것으로 보입니다.
- utils.py 쪽에서 표 셀은 bullet_count / max 컷 정도만 하고, “min 충족을 위한 후처리(padding 또는 재작성)”가 없어 실패합니다.
- 만족도 섹션은 guidelines 적용 대상에서 빠져있거나, min 강제 로직이 없어서 짧게 끝납니다.
- UI 검증은 “표도 최소 300자”, “만족도도 최소 500자”로 보고 있는데 생성물이 짧아 실패합니다.

────────────────────────────────────────
3) 제가 생각한 해결 방향(요청안)
────────────────────────────────────────
핵심은 “표 셀 + 만족도 섹션에도 min 글자수 강제 적용”입니다.
가능하면 1차 생성 후 2차 보정(짧을 때만)을 추가해서 품질을 유지하고 싶습니다.

[필수] A. 규칙(JSON) 보강
1) 표 컬럼에도 min_chars_no_space를 넣어 UI 검증과 일치시키기
- 예: part1.feedback_table.columns.problem / improvement
- 예: part2.detail_table.columns.content (요구가 있다면)
- 예: part4.feedback_summary.plan 등(요구가 있다면)
※ “표 셀 300자”가 요구사항이면 min=300으로, 아니면 최소 기준을 조정해 주세요.

2) 만족도 섹션(주관식 요약/제언)에 규칙을 명확히 추가
- satisfaction.subjective_summary: min 500 (공백 제외), max 적절히(예: 700)
- satisfaction.overall_suggestion: min 500, max 적절히(예: 700)
※ 실제 앱 내부 데이터 키에 맞춰 반영해 주세요.

[필수] B. utils.py 후처리 강화(표/만족도 min 강제)
1) _apply_table_rule()에서 col_rule의 min_chars_no_space를 읽어
- 공백 제외 글자수가 min 미만이면 자동으로 보강하여 min을 충족시킵니다.
2) 보강 방식은 placeholder 금지
- “(추가 내용 필요)” 같은 문구가 결과물에 남지 않게 해 주세요.
- 자연스러운 보강 문장 리스트(PADDING_PHRASES)를 순환 추가하거나,
- 불릿이면 각 불릿 문장을 자연스럽게 확장하는 방식으로 보강해 주세요.
3) 적용 순서
- bullet_count 맞춤 → min 충족 보강 → max 초과 시 컷 → 최종 정리

[권장] C. 2단계 생성(짧을 때만) 옵션
- 표 셀/만족도 문단이 min을 못 맞춘 경우에만,
  현재 텍스트 + 입력 요약(snippet)을 넣고 강한 프롬프트로 1~2회 재작성하여 품질을 높입니다.
- 무한 재시도 금지(최대 2회).

[필수] D. UI 정리(그림1,2 삭제 또는 기본 숨김)
- “규칙 코드 상태/작성지침 적용 로그/검증 결과” 등은
  기본 화면(미리보기)에서 제거하거나 expander/checkbox로 기본 숨김 처리해 주세요.
- 사용자가 보는 메인 화면은 ‘작성 결과 폼’ 중심으로 유지해 주세요.

────────────────────────────────────────
4) 제가 원하는 최종 상태(검증 기준)
────────────────────────────────────────
1) PDF 3개 업로드 → AI 분석 시작
2) PART1~PART4 결과가 폼/표에 채워짐
3) 표 셀(지정된 컬럼)은 공백 제외 최소 300자 충족(요구사항 기준)
4) 만족도 요약분석/종합 제언은 각각 공백 제외 최소 500자 충족
5) 불릿 규칙이 있는 항목은 불릿 줄 수 정확히 일치
6) 미리보기에서 그림1,2에 해당하는 불필요 블록은 보이지 않음(또는 기본 숨김)

────────────────────────────────────────
5) “더 좋은 대안”이 있다면 제안하고 적용해 주세요
────────────────────────────────────────
제가 생각한 방식은 “min 강제 후처리 + 필요 시 2단계 재작성”입니다.
그런데 아래 같은 대안이 더 안정적이라면 제 방식 대신 그걸로 진행해 주세요.

- 대안1: UI 검증을 ‘표는 min 검사 제외(0~max)’, 본문만 min~max 검사로 분리(요구가 허용된다면)
- 대안2: 표/만족도를 별도 생성 파이프라인으로 분리(전용 프롬프트/전용 후처리)
- 대안3: 토큰/비용을 줄이면서 품질 유지 가능한 더 좋은 전략

단, 사용자 요구가 “표 300자 / 만족도 500자 최소”로 고정이라면
결국 min 강제 로직이 들어가야 하므로 그 기준을 충족하는 방향으로 작업해 주세요.

────────────────────────────────────────
