# Major Refactoring Task: Implement Detailed TOC Structure & Granular Editing

We need to completely restructure the application's data model, AI system instruction, and Streamlit UI to match the user's detailed Table of Contents (TOC). The core requirement is **granular editing**, meaning large text blocks must be split into smaller, section-specific text areas.

# 1. Define New Data Structure (JSON Schema)
The AI output and Streamlit Session State must strictly adhere to this new, granular schema based on the TOC.

```json
{
  "part1_general": {
    "need_1_user_desire": "String (1)이용아동의 욕구 및 문제점)",
    "need_2_local_env": "String (2)지역 환경적 특성 - 통합 기술)",
    "feedback_table": [ {"area": "String", "problem": "String", "improvement": "String"} ],
    "total_review_text": "String (2)총평)",
    "satisfaction_stats": [ {"category": "String", "very_satisfied": Int, "satisfied": Int, "normal": Int, "dissatisfied": Int} ],
    "purpose_text": "String (4.사업목적)",
    "goals_text": "String (5.사업목표)"
  },
  "part2_programs": {
    "보호": { "detail_table": [], "eval_table": [] },
    "교육": { "detail_table": [], "eval_table": [] },
    "문화": { "detail_table": [], "eval_table": [] },
    "정서지원": { "detail_table": [], "eval_table": [] },
    "지역사회연계": { "detail_table": [], "eval_table": [] }
    // Note: detail_table columns = [세부영역, 프로그램명, 대상, 인원, 주기, 계획내용]
    // Note: eval_table columns = [프로그램명, 평가도구, 평가방법, 평가시기]
