# 2. Update Word Generator (`doc_utils.py`) - The Bold Parser

Standard `python-docx` does not understand `**`. You MUST implement a parser function.

## A. Create Helper Function
Add this function to `doc_utils.py`:

```python
def add_markdown_text(paragraph, text):
    """
    Parses text with **bold** markers and adds runs to the paragraph.
    Example: "Start **Bold** End" -> Run("Start "), Run("Bold", bold=True), Run(" End")
    """
    parts = text.split('**')
    for i, part in enumerate(parts):
        run = paragraph.add_run(part)
        # Even indices are normal, Odd indices are BOLD (assuming pairs)
        if i % 2 == 1:
            run.bold = True
B. Apply to Document Generation
Update create_word_document to use this function instead of paragraph.add_run(text) or cell.text = ....

For Tables (Total Review, Feedback):

Instead of cell.text = row['content'], use:

Python
cell.text = "" # Clear default

add_markdown_text(p, row['content'])
For Text Sections (Needs, Environment, Satisfaction):

Instead of doc.add_paragraph(data['text']), use:

Python

# Set alignment to Justify for professional look
