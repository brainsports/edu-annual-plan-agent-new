# Major Task: Upgrade "Satisfaction Survey" (Part 1 - 3)

The user wants to replace the simple Pie Chart with a **Horizontal Bar Chart** and add a **Detailed Analysis Table** with 10 objective questions and 1 subjective question.

# 1. Update Data Structure (`utils.py`)
- Redefine `part1_general` -> `satisfaction_survey`.
- New Schema:
  ```json
  "satisfaction_survey": {
    "questions_list": [
      {"question": "String (Extract from PDF)", "score": Float (out of 5.0)}
    ],
    "subjective_question": "String (e.g. 기타 건의사항)",
    "subjective_analysis": "String (Summary of subjective answers, Min 500 chars)",
    "overall_suggestion": "String (Comprehensive analysis & suggestions, Min 500 chars)"
  }
2. Update AI Instructions (utils.py)
Source: Instruct Gemini to Extract the 10 specific questions from the uploaded file (PDF). If exact text is missing, use standard "Community Child Center Evaluation Indicators" (Program, Facility, Staff, Food, Safety, etc.).

Data Generation: Generate realistic satisfaction scores (e.g., 4.2, 4.8) for each of the 10 questions.

Text Analysis:

subjective_analysis: Summarize the subjective feedback in detail (Min 500 Korean chars).

overall_suggestion: Provide a comprehensive analysis and future suggestions based on the scores (Min 500 Korean chars).

3. Update Frontend UI (main.py)
A. Visual: Horizontal Bar Chart
Use st.bar_chart or altair to create a Horizontal Bar Chart.

Y-axis: Question Text.

X-axis: Score (0 to 5).

Add a title: "문항별 만족도 결과".

B. Data Table
Display the survey results in a clean table (Question | Score).

Use st.dataframe or st.table.

C. Text Analysis Area
Create 2 large text areas (or use st.data_editor with strict width if preferred, but text_area is better here for simple text).

1. 주관식 문항 분석: st.text_area("주관식 문항 요약 및 분석 (500자)", value=..., height=300)

2. 종합 분석 및 제언: st.text_area("종합 분석 및 제언 (500자)", value=..., height=300)

4. Update Word Generator (doc_utils.py)
Render the Satisfaction Table (Question | Score) in the Word doc.

Render the Two Analysis Paragraphs clearly.

(Optional) If chart generation for Word is too complex, ensure the Table and Text are formatted perfectly with "Justified" alignment.

Action
Apply these changes to utils.py and main.py immediately.
