# Visual Polish Task: Finalize Word Document Styling (Margins, Widths, & Full Justification)

The user requires a professional layout for the generated Word document (.docx). We need to strictly update `doc_utils.py` to handle margins, table dimensions, and text alignment (including INSIDE tables).

# 1. Page Margins: Set to "Narrow" (0.5 inch)
- In the document initialization function:
  ```python
  from docx.shared import Inches
  section = document.sections[0]
  section.left_margin = Inches(0.5)
  section.right_margin = Inches(0.5)
  section.top_margin = Inches(0.5)
  section.bottom_margin = Inches(0.5)
2. Global Alignment Rule: Justified (양쪽 정렬)
Body Text: All standard paragraphs added to the document must be set to WD_ALIGN_PARAGRAPH.JUSTIFY.

Table Text: CRITICAL. After populating ANY table (Feedback Table, Total Review Table, etc.), iterate through every cell and force paragraph.alignment = WD_ALIGN_PARAGRAPH.JUSTIFY.

3. Table Column Widths (Fixed Ratios)
You must disable autofit (table.autofit = False) and set explicit widths. Assuming ~7.5 inches of usable width:

A. Feedback Plan Table (환류 계획)
Target Ratio: 20% : 40% : 40%

Logic:

Column 0 (Area): 1.5 Inches

Column 1 (Problem): 3.0 Inches

Column 2 (Improvement): 3.0 Inches

B. Total Review Table (총평)
Target Ratio: 20% : 80%

Logic:

Column 0 (Category): 1.5 Inches

Column 1 (Content): 6.0 Inches

Implementation Hint
When creating tables in doc_utils.py:

Create table -> table.autofit = False

Set column widths logic.

Fill data.

Final Loop:

Python
for row in table.rows:
    for cell in row.cells:
        for paragraph in cell.paragraphs:
            paragraph.alignment = WD_ALIGN_PARAGRAPH.JUSTIFY
Action
Update doc_utils.py to apply these strictly formatted styles.
