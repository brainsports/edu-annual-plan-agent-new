참참 4.3 연간사업계획서 생성도우미 작업 지침
1. 프로젝트 개요 (반드시 이해해야 할 큰 목표)

이 프로젝트는 참참 4.3 연간사업계획서 생성도우미이다.

목표는 다음과 같다.

지역아동센터에서 사용한 프로그램 평가서 PDF를 업로드하면

AI가 내용을 자동으로 분류·추출·정리하고

그 결과를 바탕으로

연간사업계획서 Part 1

연간사업계획서 Part 2

상반기 / 하반기 계획서 (월간 계획 표 포함 가능)

을 단계별 UI 흐름으로 생성·편집하게 하며

최종적으로 Word(.docx) 다운로드까지 이어지는 완성형 문서 생성 앱이다.

이 앱은 “한 번에 자동 생성”이 아니라
업로드 → 분석 → 초안 생성 → 사용자 편집 → 최종 다운로드 구조를 가진다.

2. 앱 전체 흐름 (Step 기반 UX 구조)

화면 상단에는 Step Indicator가 있으며, 아래 순서를 정확히 따른다.

PDF 업로드

자동 분류 / 분석

연간 Part 1

연간 Part 2

상반기 계획

하반기 계획

다운로드

즉, 이 앱의 핵심 구조는 다음과 같다.

업로드 → 분석/정리 → 문서 초안 생성(탭별 편집) → 최종 다운로드

각 Step은 독립된 Page 컴포넌트를 가지며,
이전 단계에서 생성된 데이터는 Zustand store를 통해 다음 단계로 전달된다.

👉 각 Page는
해당 단계에서 필요한 입력 UI 구성 + store 데이터 연결 + 미리보기 렌더링만 담당한다.

👉 Page 안에 UI 로직을 직접 만들지 말고,
가능하면 이 폴더의 컴포넌트를 활용하거나 확장한다.

📂 client/src/lib/

공통 로직 / 인프라 영역

store.ts → Zustand 상태 관리

exportDocx.ts → Word 다운로드 처리

utils.ts → 공통 유틸

queryClient.ts → API 통신 관련

📂 shared/schema/

이 프로젝트의 절대 기준

AnnualPlan 타입

Part1 / Part2 / 월간계획 관련 타입

서버/클라이언트 공용 스키마

⚠️ 모든 데이터 구조는 shared/schema 기준을 절대적으로 따른다.
UI를 위해 타입을 바꾸지 않는다.
필요하면 기본값 생성 함수로 해결한다.

5. 현재 상태 요약 (중요)

TypeScript 오류(AnnualPlan 필수 필드 누락)는
→ 기본값(id, title, createdAt 등)을 생성하는 방식으로 해결됨

GitHub 리포지토리 생성 및 Push 완료
→ 새 창에서도 이어서 작업 가능한 “안전한 저장 상태”

그러나
→ AnnualPlanPart1Page.tsx에서 UI 렌더링 일부가 사라진 상태

원인 추정:

타입 오류 해결 과정에서

섹션 정의

map 렌더링

입력 UI 연결
중 일부가 제거되었을 가능성이 큼

7. 작업 시 절대 지켜야 할 원칙

shared/schema 타입을 바꾸지 않는다

UI 문제는 렌더링/구조 문제로 해결한다

한 번에 여러 단계를 건드리지 않는다

항상 “지금 단계에서 사용자가 무엇을 편집하는가” 기준으로 판단한다

Page 컴포넌트는 복잡해져도 괜찮지만, 흐름은 단순해야 한다
