[목표]
Streamlit 기반 “AI 연간 사업계획 통합 에이전트”에서 반복 발생하는 오류들을 영구적으로 막고, main.py ↔ utils.py ↔ doc_utils.py 간 인터페이스를 안정화한다.
특히 다음 오류들이 다시는 발생하지 않게 수정한다.
- TypeError: get_partitioned_analysis() got an unexpected keyword argument 'progress_callback'
- TypeError: get_partitioned_analysis() got an unexpected keyword argument 'month_bucket'
- AttributeError: 'UploadedFile' object has no attribute 'get'
- ImportError: cannot import name 'generate_part1_report' from 'doc_utils'

[반드시 지킬 원칙]
1) main.py의 호출부는 가능한 바꾸지 말고(변경 최소화), utils.py가 다양한 호출 인자를 “수용”하도록 한다.
2) 함수 시그니처 불일치로 앱이 죽지 않도록, utils.py에서 **kwargs 안전 수용을 기본값으로 만든다.
3) UploadedFile(스트림릿)과 dict(내부 데이터)를 모두 받을 수 있게 타입 호환을 보장한다.
4) doc_utils.py와 utils.py는 서로 다른 파일이다. 역할을 분리하고, import 대상 함수가 doc_utils.py에 실제로 존재하도록 맞춘다.
5) 수정 후 Preview에서 업로드 → 분석 → 월별 미리보기까지 에러 없이 동작해야 한다.

------------------------------------------------------------
[수정 1] utils.py: get_partitioned_analysis 시그니처/호환성 강화 (핵심)
- utils.py의 get_partitioned_analysis(...)를 아래 조건을 만족하도록 수정한다.
  a) progress_callback, month_bucket 인자를 지원한다.
  b) 추가로 앞으로 어떤 키워드가 더 들어와도 죽지 않게 **_ignored_kwargs 를 받는다.
  c) progress_callback은 None일 수 있으며, 호출할 때 안전하게 감싼다(예: _safe_progress).
  d) month_bucket이 dict로 넘어오면, 분석 결과에서 월별 버킷을 계산하여 month_bucket.clear(); month_bucket.update(...) 로 채워준다.
- get_partitioned_analysis는 최종적으로 analysis(dict)를 return 한다.

------------------------------------------------------------
[수정 2] utils.py: UploadedFile/dict 혼용 입력 처리 (AttributeError 방지)
- extract_file_summaries(files_data)에서 item이 dict인 경우와 UploadedFile인 경우를 모두 처리하도록 한다.
- UploadedFile은 dict처럼 .get()을 쓰면 안 된다. 반드시 getattr(item, "name"), getattr(item, "type"), item.getvalue()/item.read() 방식으로 처리한다.
- read_uploaded_file(uploaded_file) 역시 getvalue() 실패 시 read()로 폴백하도록 한다.
- main.py에서 uploaded_files를 그대로 넘겨도 요약 단계에서 절대 죽지 않게 한다.

------------------------------------------------------------
[수정 3] doc_utils.py: main.py에서 import 하는 함수들이 실제로 존재하도록 정리 (ImportError 방지)
- main.py에서 다음을 import 하고 있다면, doc_utils.py에 동일한 이름의 함수가 반드시 존재해야 한다:
  generate_part1_report, generate_part2_report, generate_monthly_report, generate_monthly_program_report, generate_full_report
- 만약 실제 구현이 아직 없다면, 최소한 “동작하는 스텁(placeholder)”이라도 제공해서 ImportError가 나지 않게 한다.
- 스텁은 입력을 받아 간단한 docx 파일(또는 bytes)을 생성/반환하는 형태로 통일한다.
  (예: python-docx 사용, 최소 제목/섹션/내용만 넣어도 됨)
- 반환 타입은 main.py에서 기대하는 형태로 맞춘다(예: 파일 경로 or bytes). 프로젝트 현재 방식에 맞춰 통일한다.

------------------------------------------------------------
[수정 4] main.py: 호출부 최소 정리 (필요 시에만)
- main.py에서 get_partitioned_analysis 호출 시 progress_callback=..., month_bucket=... 등을 넘기더라도 utils.py가 받도록 했으므로 main.py는 원칙적으로 변경하지 않는다.
- 단, 만약 main.py가 dict/list 변환을 잘못해 utils.py에 불필요한 형태를 넘긴다면 그 부분만 최소 수정한다.
- 업로드 파일 수/총 용량 제한 로직은 유지한다.

------------------------------------------------------------
[검증 체크리스트]
1) Preview 실행 시 ImportError 없이 화면이 로드된다.
2) PDF 3개 이상 업로드 후 “AI 분석 시작” 클릭 시:
   - JSON 생성 단계에서 에러 없이 완료된다.
   - progress_callback을 넘겨도 TypeError가 나지 않는다.
   - month_bucket을 넘겨도 TypeError가 나지 않는다.
3) UploadedFile 처리 과정에서 .get() 관련 AttributeError가 절대 발생하지 않는다.
4) 월별 프로그램 배치 미리보기(expander/섹션)가 정상적으로 표시된다.
5) 수정 후 코드 스타일/중복 정의/미사용 함수가 있다면 정리한다(단, 기능 우선).
