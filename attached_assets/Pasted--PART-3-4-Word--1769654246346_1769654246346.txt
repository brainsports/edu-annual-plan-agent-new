✅ 레플릿 수정 요청 프롬프트 (PART 3/4 Word 출력 수정 + 원인보고 필수)

문제를 해결하려고 하는데 내 생각은 아래와 같아. 네가 더 좋은 대안이 있으면 그렇게 하고, 없으면 요청대로 작업해줘.

0) 전제

PART 1, PART 2는 이미 완료된 상태이므로 절대 건드리지 마.

이번 수정은 Word(DOCX) 출력물 기준이다.

목표는 다음 3가지가 반드시 반영되는 것이다.

1) PART 3(1~6월 월별 사업계획) 수정 요구사항
(A) “월별 페이지 분리”가 반드시 되어야 함

현재는 한 페이지에 1월 표 아래로 다음 달 표가 이어져 들어가면서 월별 페이지 분리 실패.

요구: 각 월(1월~6월)은 무조건 ‘새 페이지’에서 시작해야 한다.

구현 규칙:

1월 테이블이 끝나면 add_page_break()

2월 테이블이 끝나면 add_page_break()

…

6월은 마지막이므로 마지막 월 뒤에는 page break를 넣지 않아도 됨

주의: “월 제목(예: 2월)”도 새 페이지 맨 위에서 시작해야 한다(표만 나뉘면 실패).

(B) “가로탭(열 너비) 비율”을 가독성 있게 조정

요구: ‘프로그램명’ 열 + ‘사업내용’ 열 폭을 키우고, 나머지 열은 균등 분배

구현 규칙(권장):

테이블 autofit 끄기(폭 고정)

총 폭 기준으로:

프로그램명: 넓게(예: 2배 수준)

사업내용: 가장 넓게(예: 2.5~3배 수준)

나머지(대분류/중분류/참여자/수행인력 등): 동일 폭

“대충 넓힌 것처럼 보임”이 아니라, 출력물에서 확실히 폭 차이가 체감되어야 한다.

2) PART 4(하반기 7~12월 월별 사업계획 및 평가) 수정 요구사항 — 매우 중요
(A) “예산계획” + “환류요약” 섹션을 출력에서 완전 삭제

현재 출력물에 예산계획/환류요약이 남아있어서 삭제 실패.

요구: PART 4 Word 결과물에서 다음은 아예 생성하지 말 것

예산계획 제목/문단/표 전체

환류 요약 제목/문단/표 전체

문서 중간/마지막 어디에도 나오면 실패.

(B) “월별 페이지 분리”가 반드시 되어야 함

PART 4에서도 7월~12월이 각 월이 새 페이지에서 시작해야 한다.

구현 규칙은 PART 3과 동일:

7월 끝나면 page break

8월 끝나면 page break

…

12월은 마지막이므로 뒤에 page break 생략 가능

월 제목도 새 페이지 상단에서 시작.

(C) “가로탭(열 너비) 비율” PART 3과 동일 적용

PART 4의 월별 표도:

프로그램명/사업내용 폭 확대

나머지 균등

3) 어디를 고칠지(반드시 코드 위치를 찾아서 “정확히” 수정)

레포 안에서 Word 생성하는 부분(대부분 doc_utils.py 혹은 Word 생성 관련 모듈)에 다음이 있을 것이다:

PART 3 Word 생성 함수(월별 루프)

PART 4 Word 생성 함수(월별 루프 + 예산계획/환류요약 생성 블록)

해야 할 일:

PART 3: 월 루프에서 월 단위로 제목+표 생성 후 page break 삽입

PART 3: 표 생성 직후 열 너비 고정 로직 적용

PART 4: 예산계획/환류요약 생성하는 코드 블록을 완전 제거하거나 실행 경로에서 제외

PART 4: 월 단위 page break + 열 너비 고정 로직 동일 적용

4) “왜 3번이나 요청했는데 수정이 안 됐는지” 원인 분석 보고(필수)

아래 질문에 반드시 답해라. 답 없이 “수정했습니다”만 하면 작업 실패로 본다.

이전 시도에서 왜 결과물이 그대로였나?

(예: 수정한 함수가 실제로 호출되지 않음 / 다른 파일이 Word 생성 담당 / 캐시/빌드 문제 / 테스트 루트가 다름 / 조건문 때문에 코드가 실행 안 됨 등)

이번에는 어떤 파일/함수/코드 블록을 수정했고, 그 근거는 무엇인가?

같은 문제가 재발하지 않게 하려면 어떤 확인 절차가 필요한가?

5) 자체 테스트를 반드시 실시해줘 (아주 중요)

수정 후 샘플/실데이터로 실행 테스트, OK/NG와 근거 제출, 실패 시 원인 수정 후 재테스트 반복.

테스트 체크리스트(OK/NG로 보고)

[PART 3]

(1) 1월~6월이 월마다 새 페이지에서 시작하는가? (OK/NG + 근거)

(2) “프로그램명”과 “사업내용” 열이 눈에 띄게 넓어졌는가? (OK/NG + 근거)

[PART 4]

(3) “예산계획” 표/제목이 완전히 사라졌는가? (OK/NG + 근거)

(4) “환류요약” 표/제목이 완전히 사라졌는가? (OK/NG + 근거)

(5) 7월~12월이 월마다 새 페이지에서 시작하는가? (OK/NG + 근거)

(6) 열 너비 비율이 PART 3과 동일하게 적용되는가? (OK/NG + 근거)

근거는 “말”이 아니라,

생성된 DOCX를 열어 확인한 결과 요약 +

코드에서 page break / column width 적용한 부분(함수명/라인 근처 설명)
로 제시해라.

6) 작업 완료 보고 형식(이대로 제출)

수정한 파일 목록

수정한 함수/코드 블록 설명

테스트 결과(위 체크리스트 OK/NG)

이전에 반영이 안 됐던 원인 3가지 이상 + 이번 해결 방식

원하는 결과는 “대충 비슷”이 아니라, 첨부 스크린샷 기준으로 월별 페이지 분리 + 열 너비 조정 + PART 4 예산/환류 삭제가 확실히 보이는 출력물이다.
