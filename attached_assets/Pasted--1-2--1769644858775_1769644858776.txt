[프로젝트/상황]
- 화면(그림1~2)에서 만족도조사 문항이 10문항이어야 하는데 3문항만 생성/표시되고 있음.
- 출력물(워드)에서도 만족도조사 표가 3문항만 나오고, 그래프가 아예 출력되지 않음(그림6).
- 워드 출력 표의 가로 폭(컬럼 비율)이 요구 비율대로 적용되지 않음.
  - 전년도 사업평가 및 환류계획 표: 영역20% / 문제점40% / 개선방안40% (그림3~4)
  - 총평 표: 영역20% / 내용80% (그림5)
  - 만족도 문항별 응답 분포 표: 문항60% / 그외40%를 6칸(5점,4점,3점,2점,1점,평균) 균등 분할 (그림8)
- 만족도 그래프는 그림7처럼 “막대그래프 2개(항목별 평균 점수 / 응답 분포(인원수))”가 워드에 포함되어야 함.

[최종 목표(완료 기준)]
1) 만족도조사 문항은 항상 10문항으로 생성/표시/출력된다.
   - 샘플 데이터 로드/실제 PDF 분석/AI 생성 모두 동일하게 10문항을 유지한다.
   - 현재처럼 3문항만 생성되는 현상이 재발하지 않는다.
2) 만족도 그래프가 워드 출력물에 반드시 포함된다.
   - 그림7과 같이 2개의 그래프(항목별 평균 점수, 응답 분포(인원수))가 출력된다.
   - 그래프가 누락되더라도 최소한 “왜 누락됐는지”가 로그에 남고, 문서 생성은 실패하지 않는다(표는 우선 출력).
3) 워드 출력 표의 컬럼 폭 비율이 요구사항대로 고정된다(자동 폭 조정으로 깨지지 않게).
   - 전년도 환류계획 표: 20/40/40
   - 총평 표: 20/80
   - 만족도 분포 표: 문항 60 / (5점~평균 6칸) 각 6.666…%씩(=40/6) 균등
4) 기존 다운로드 UX 규칙은 건드리지 않는다.
   - “전체 다운로드” 금지, 파트별 다운로드 유지(기존 규칙 유지).

[우선순위(이 순서대로 처리)]
A. “10문항이 3문항으로 줄어드는 원인”을 먼저 잡는다(데이터 생성/파싱/표시 어디서 잘리는지).
B. 워드 출력에서 그래프가 누락되는 원인을 잡고, 누락 방지(필수 삽입 + 실패 시 로그/대체 처리).
C. 워드 표 컬럼 폭 비율을 정확히 고정하는 유틸을 만들고, 해당 표들에 일괄 적용한다.
D. 샘플 데이터/실데이터/AI 생성 전 경로에서 동일하게 재현 테스트한다.

[작업 지침(꼼꼼 버전)]
1) 재현 체크(반드시 기록)
- “샘플 데이터 로드” 버튼 → 만족도조사 섹션 진입 → 문항이 10개인지 확인(현재는 3개)
- 같은 상태에서 워드 출력 생성 → 만족도 그래프가 들어가는지 확인(현재는 누락)
- 위 과정을 콘솔/서버 로그에 남겨서, 문항 수가 어느 단계에서 3으로 줄어드는지 확인

2) 만족도 문항 10개 강제 규칙(프론트 + 백엔드/문서 모두)
- 프로젝트 전체에서 만족도 문항 수를 상수로 고정:
  - SURVEY_QUESTION_COUNT = 10 (혹은 동일 의미 상수)
- “문항 데이터”가 어디서 오든(샘플/AI/파싱) 최종 사용 전 아래 보정 로직을 반드시 통과:
  (1) 문항 배열 길이가 10 미만이면 → 부족한 문항을 자동으로 채워 10개로 만든다.
      - 채우는 방식은 두 가지 중 하나로 선택:
        a) (권장) 문항 텍스트 기본 템플릿으로 4~10번을 자동 생성
        b) AI가 10개를 반환하도록 프롬프트를 강화하고, 그래도 부족하면 a)로 보정(안전장치)
  (2) 문항 배열 길이가 10 초과이면 → 상위 10개만 사용(문서/표/차트 일관성)
  (3) 응답 분포(5~1점)도 10문항 길이에 맞춰 동기화:
      - 누락된 문항은 0으로 채움 또는 균등 분배(샘플은 0이 안전)
      - 평균 계산도 10문항 기준으로 재계산
- 이 규칙을 “UI 표시”뿐 아니라 “워드 생성 직전”에도 한 번 더 적용(이중 안전장치)

3) AI 생성 프롬프트/파싱 문제(3문항만 생성되는 원인 제거)
- AI가 3문항만 돌려주는 경우가 있다면, 프롬프트를 아래처럼 강화:
  - “만족도 문항은 정확히 10문항, 1~10번까지 번호를 포함”
  - “각 문항별 5점~1점 응답 인원 합은 총응답인원과 일치”
  - “JSON은 10문항 배열로 고정”
- 파싱에서 3문항만 남는다면, 아래를 점검:
  - (a) JSON schema/타입 정의에서 배열 길이를 3으로 제한하는 코드가 있는지
  - (b) slice(0,3) / filter 조건 / UI map에서 3개만 렌더링하도록 제한한 코드가 있는지
  - (c) 샘플 데이터 상수가 3개로만 정의돼 있는지(샘플 데이터를 10개로 교체)
- 결론: “원인 제거 + 10문항 보정 로직”을 둘 다 적용해서 재발 방지

4) 워드 출력: 표 컬럼 폭 비율 고정(핵심)
- python-docx(혹은 워드 생성 라이브러리)에서 표 폭이 자동으로 깨지지 않도록 아래 원칙 적용:
  - table.autofit = False (또는 autoFit 비활성화)
  - 문서 usable_width(페이지폭-좌우마진)을 계산해서, 비율대로 폭을 “실제 길이”로 할당
- 권장 구현(유틸 함수로 만들어 재사용):
  - set_table_width_by_ratio(table, ratios, usable_width)
  - 내부에서 각 컬럼의 모든 셀에 동일 width를 반복 적용(셀 단위로 적용 필요)
- 적용 대상 표 3개(반드시 모두 적용):
  (1) 전년도 사업평가 및 환류계획 표(3열):
      - [영역, 문제점, 개선방안] = [0.20, 0.40, 0.40]
  (2) 총평 표(2열):
      - [영역, 내용] = [0.20, 0.80]
  (3) 만족도 문항별 응답 분포 표(7열):
      - [문항] = 0.60
      - [5점,4점,3점,2점,1점,평균] = 0.40/6씩(각 0.066666…)
- “표가 페이지 밖으로 튀는지”, “셀 줄바꿈/문단 들여쓰기 때문에 폭이 깨지는지”도 함께 점검
  - 필요 시 셀 내부 단락의 좌우 들여쓰기/셀 마진을 최소화

5) 워드 출력: 만족도 그래프 누락 문제 해결(매우 심각)
- 현재 UI에서는 그래프가 보이는데 워드에서는 누락되는 것으로 보임 → 원인은 보통 아래 중 하나:
  1) 그래프 이미지 파일 생성 실패(경로/권한/라이브러리 오류)
  2) 생성은 되었는데 doc.add_picture 단계가 호출되지 않음(조건 분기/예외로 스킵)
  3) 파일 생성 후 즉시 삭제/경로가 바뀜(/tmp 사용 시 타이밍 문제)
  4) 이미지 크기/비율 설정 오류로 0 크기처럼 들어가거나 깨짐
- 해결 원칙:
  - (A) 차트 생성 함수에서 예외 발생 시 반드시 로그 남김(스택트레이스)
  - (B) 워드 삽입 직전 파일 존재 여부 확인 + 없으면 에러 로그 + “차트 생성 실패 안내 문구”라도 문서에 남김(문서 생성은 계속)
  - (C) 차트 2개는 그림7처럼 나란히 배치:
      - 2열짜리 표를 만들어 각 셀에 그래프 이미지를 넣는 방식 권장(레이아웃 안정)
      - 각 이미지 폭은 usable_width의 0.48~0.50 정도로 맞추고 높이는 자동
  - (D) 차트 생성 후 plt.close() 등 리소스 정리로 누락/빈 이미지 방지
- 테스트 기준:
  - 샘플 데이터 로드 → 워드 출력 → 그래프 2개가 반드시 포함
  - 실제 업로드 데이터 → 워드 출력 → 그래프 2개 포함

6) 수정해야 할 파일 찾는 방법(에이전트가 직접 탐색)
- 프로젝트에서 아래 키워드로 전수 검색해서 “문항 3개 제한/슬라이스/샘플상수”를 찾고 제거:
  - "만족도", "satisfaction", "survey", "문항", "question", "distribution", "chart", "plt", "add_picture"
- 특히 워드 생성 관련 파일(예: doc_utils.py, utils.py, main.py 등)에서
  - 만족도 섹션 생성 함수(표/차트 생성) 위치를 찾아 수정
- 프론트(UI)에서는
  - 만족도 문항 렌더링하는 컴포넌트/페이지에서 map이 3개만 돌도록 제한된 코드가 있는지 확인
  - 샘플 데이터(문항 3개짜리 상수)가 있으면 10개로 교체

7) 작업 결과 보고 방식(반드시 이렇게)
- 변경한 파일 목록 + 각 파일에서 무엇을 왜 바꿨는지 요약
- 재현/검증 체크리스트 결과(OK/NG) 함께 제출:
  - [ ] UI 만족도 문항 10개 표시
  - [ ] 워드 만족도 표 10문항 출력
  - [ ] 워드 만족도 그래프 2개 출력(그림7 형태)
  - [ ] 전년도 환류계획 표 20/40/40 적용
  - [ ] 총평 표 20/80 적용
  - [ ] 만족도 분포 표 문항60 + 나머지 6칸 균등(40/6) 적용
  - [ ] 기존 다운로드 UX 규칙 유지(전체 다운로드 없음)

[추가 제약]
- 기능을 “임시로 숨기기”가 아니라, 원인을 제거해서 재발하지 않게 수정해줘.
- 표 폭/그래프 출력은 “샘플 데이터”와 “실제 업로드 데이터” 둘 다에서 동일하게 동작해야 해.
