1) 문제 요약

사용자 화면에 아래 “내부용(디버그/검증/로그)” UI가 노출되고 있다. 운영(Cloud Run/WordPress 공개)에서는 절대 노출되면 안 된다.

사이드바: 디버그 정보 / 업로드된 내용 미리보기 / 월별 프로그램 배치 미리보기 / 작성지침 규칙(JSON)

본문: 규칙 검증 결과(디버그), 규칙 로드 상태, 작성지침 적용 로그, JSON/로그/규칙 텍스트 출력 전부

우측 상단 메뉴(점 3개): Rerun / Settings / Print / Record a screencast 메뉴도 숨김 필요

2) 목표(운영 기준)

운영 기본값: 위 항목이 모두 숨겨진다.

개발/점검이 필요할 때만: 특정 조건에서만 내부 메뉴가 보이도록 토글한다.

권장: 환경변수 SHOW_INTERNAL=1일 때만 보이게 한다.

(선택) 추가: URL 쿼리 ?debug=1이면 보이게 허용한다(단, 운영에서는 기본 숨김 유지).

3) 구현 지시(반드시 적용)
A. “내부 메뉴 표시 여부” 공통 스위치 만들기

main.py 상단에 아래와 같은 전역 플래그를 만든다.

우선순위 예시: SHOW_INTERNAL 환경변수 → (선택) debug=1 쿼리

이 플래그를 기준으로 내부용 UI 블록을 통째로 감싼다.
(중요: 텍스트 일부만 숨기는 게 아니라, 해당 expander/섹션 자체를 렌더링하지 않게 한다.)

예시(참고 코드):

import os
import streamlit as st

def is_internal_enabled():
    env_on = os.getenv("SHOW_INTERNAL", "0").strip() == "1"
    # 쿼리 파라미터는 streamlit 버전에 따라 다르니 안전하게 처리
    try:
        qp = dict(st.query_params)
        debug_on = str(qp.get("debug", "0")) in ["1", "true", "True"]
    except Exception:
        debug_on = False
    return env_on or debug_on

SHOW_INTERNAL = is_internal_enabled()

B. 숨김 대상 UI(필수)

사이드바의 아래 항목을 모두 if SHOW_INTERNAL:로 감싼다.

“디버그 정보”

“업로드된 내용 미리보기”

“월별 프로그램 배치 미리보기”

“작성지침 규칙(JSON)”

본문(메인 영역)의 아래 항목도 모두 if SHOW_INTERNAL:로 감싼다.

“규칙 검증 결과(디버그)”

“규칙 로드 상태”

“작성지침 적용 로그”

규칙/로그/JSON이 그대로 출력되는 모든 텍스트

내부 검증은 서버 로그(logging) 로 남기고, 사용자 화면에는 노출하지 않는다.

C. 우측 상단(점 3개) Streamlit 메뉴 숨김(필수)

.streamlit/config.toml에서 툴바 모드를 최소화한다.

client.toolbarMode = "minimal" 또는 "viewer" 중 더 잘 숨겨지는 것으로 선택한다.

CSS로 한 번 더 확실히 숨긴다(버전 차이 대비).

HIDE_STREAMLIT_UI = """
<style>
#MainMenu {visibility: hidden;}
header {visibility: hidden;}
footer {visibility: hidden;}
[data-testid="stToolbar"] {visibility: hidden;}
[data-testid="stDecoration"] {visibility: hidden;}
</style>
"""
st.markdown(HIDE_STREAMLIT_UI, unsafe_allow_html=True)

5) 왜 이전에 3번 요청했는데 반영이 안 됐는지 “원인 답변 필수”

다음 중 무엇 때문이었는지 정확히 답변해 달라.

실제 실행 경로에서 호출되는 파일/함수가 달랐는지(main.py가 아니라 다른 파일/분기였는지)

캐시/리로드 문제로 구버전 UI가 다시 살아난 건지

조건문/분기 때문에 특정 상황에서만 숨김 코드가 실행되지 않았는지

Word 생성 로직/화면 렌더링 로직이 분리되어 있는데 잘못된 쪽만 수정했는지

6) 자체 테스트 요구(필수)

수정 후 샘플/실데이터로 실행 테스트를 반드시 하라.

운영 모드(SHOW_INTERNAL=0)에서: 내부 메뉴가 완전히 숨겨짐을 캡처로 증명한다(OK/NG와 근거 제출).

내부 모드(SHOW_INTERNAL=1 또는 ?debug=1)에서: 내부 메뉴가 정상적으로 나타남을 캡처로 증명한다(OK/NG와 근거 제출).

실패 시 원인을 수정하고 재테스트를 반복한다.
