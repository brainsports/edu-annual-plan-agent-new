문제를 해결하려고 하는데 내 생각은 아래와 같아.
네가 더 좋은 대안이 있으면 그렇게 하고, 없으면 요청대로 작업해줘.

또한 수정 후 반드시 자체 테스트를 실시해라.
- 샘플 데이터 및 실제 업로드 데이터로 실행 테스트
- 각 단계별 OK / NG 판단 및 근거 제시
- 실패 시 원인 수정 후 재테스트를 반복한다.

────────────────────────
[1. 수정 목표 – 가장 중요]
────────────────────────
현재 앱은 기술적 요소가 과도하게 노출되어 사용자(사회복지사, 비전공자)에게
“무섭고 복잡한 화면”으로 인식된다.

이를 다음 원칙으로 전면 수정한다.

- 미니멀리즘 기반
- 친절하고 따뜻한 UX
- 한 화면 = 한 행동
- 기술 용어/디버그 요소 완전 차단
- ‘업무 도구’가 아닌 ‘도와주는 비서’ 느낌

기능은 유지하되, **보이는 UI만 전면 재설계**한다.

────────────────────────
[2. 디자인 톤 & 컬러 기준]
────────────────────────
- 메인 컬러: 부드러운 퍼플 계열 (#7B6CF6 또는 유사 톤)
- 보조 컬러: 연한 블루, 웜 오렌지 (포인트용)
- 배경: 완전 흰색 ❌ → 아주 연한 오프화이트/라이트 그레이
- 빨강/경고색 최소화 (에러도 부드러운 안내 문구로 대체)

- 카드형 UI 사용
- 둥근 모서리
- 충분한 여백
- 그림자 최소화

────────────────────────
[3. 첫 화면(UI) 구조 전면 개편]
────────────────────────
앱 최초 진입 시 화면을 아래 구조로 고정한다.

[상단]
- 타이틀:
  “AI 사업계획 도우미”
- 서브 문구(작게):
  “파일을 올리면 찹찹 AI가 연간 사업계획 작성을 도와드려요.”

[중앙 메인 카드 – 가장 중요]
- 큰 카드 1개만 표시
- Drag & Drop 파일 업로드 영역
- 문구:
  “사업 보고서 파일을 올려주세요”
- 보조 설명:
  “PDF / DOCX 파일 업로드 가능”
- 메인 버튼:
  “파일 올리기”
  (퍼플 컬러, 둥근 버튼)

[친근한 캐릭터 영역]
- 업로드 카드 우측 또는 하단에
  문서를 들고 있는 친근한 캐릭터 일러스트 배치
- 말풍선:
  “걱정 마세요. 제가 차근차근 도와드릴게요 🙂”

[진행 과정 안내]
- 아이콘 + 짧은 문구 3단계 고정 표시
  ① 파일 업로드
  ② AI 분석
  ③ 결과 확인 & 다운로드

────────────────────────
[4. 버튼 및 문구 UX 전면 수정]
────────────────────────
❌ 기존 기술적 표현 전부 제거

- “AI 분석 시작” → “AI가 분석 시작하기”
- “JSON 생성 중” → “내용을 정리하고 있어요…”
- “규칙 검증 결과” → 사용자에게 노출 ❌
- “디버그 정보” → 사용자에게 노출 ❌

상태 메시지 예시:
- “파일을 읽고 있어요…”
- “사업계획을 정리 중이에요…”
- “거의 다 됐어요. 잠시만 기다려 주세요.”

완료 메시지:
- “분석이 끝났어요! 결과를 확인해 보세요.”

────────────────────────
[5. 반드시 숨겨야 할 요소 (중요)]
────────────────────────
아래 항목은 **일반 사용자 화면에서 완전히 제거**한다.
CSS 숨김 ❌, 조건부 렌더링으로 아예 출력 금지 ⭕

- 디버그 정보 패널
- 업로드된 내용 미리보기
- 월별 프로그램 배치 미리보기
- 규칙 검증 결과(디버그)
- JSON 원문 / 로그 / 규칙 전문
- 우측 상단 Rerun / Settings / Print / Record 메뉴

구현 방식:
- production 환경에서는 절대 렌더링하지 않는다.
- 개발자 전용 플래그(is_dev / is_admin)가 있을 때만 내부 접근 허용

────────────────────────
[6. 샘플 데이터 UX 정리]
────────────────────────
- “샘플 데이터 로드” 버튼 유지
- 메인 버튼보다 작은 크기
- 중립 색상
- 설명 문구:
  “처음이라면 샘플로 먼저 체험해 보세요.”

────────────────────────
[7. 파일 형식 관련 UX 정리]
────────────────────────
- HWP 관련 안내 문구 완전 삭제
- 지원 불가 형식은 조용히 무시하거나
  “지원하지 않는 파일 형식이에요” 정도의 부드러운 안내만 표시

────────────────────────
[8. 기술 구현 원칙]
────────────────────────
- 로직 변경 ❌
- UI / 컴포넌트 구조 / 렌더링 방식만 수정
- 기존 함수, API, 분석 흐름은 그대로 유지
- 조건부 렌더링으로 사용자/개발자 화면 분리

────────────────────────
[9. 완료 기준 (필수)]
────────────────────────
- 처음 접속한 사회복지사가
  설명 없이도 “파일 올리면 되는구나”를 바로 이해해야 한다.
- 화면을 봤을 때 부담감이 없어야 한다.
- 기술 용어를 몰라도 사용 가능해야 한다.
- 테스트 후 실제 사용자 화면에서
  디버그/JSON/규칙 관련 요소가 단 하나도 보여서는 안 된다.
