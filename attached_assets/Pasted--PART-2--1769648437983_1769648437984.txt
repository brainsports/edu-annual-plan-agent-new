문제를 해결하려고 하는데 내 생각은 아래와 같아. 네가 더 좋은 대안이 있으면 그렇게 하고, 없으면 요청대로 작업해줘.

[수정 범위]
- PART 2: 세부 사업 계획
- 문서형태 미리보기(UI) + 워드 출력(docx)
- 평가계획 표(5열: 세부영역/프로그램명/기대효과/평가계획/평가방법)

────────────────────────────────────────
[1] 문서형태 미리보기: 기대효과가 안 보이는 문제(필수)
[문제]
- 문서형태 미리보기에서 ‘계획내용’은 보이는데 ‘기대효과’가 표시되지 않는다.

[요구사항]
- 문서형태 미리보기에서 각 항목마다 아래 2줄을 항상 보이게 한다.
  1) 기대효과: (내용)
  2) 계획내용: (내용)
- ‘기대효과’와 ‘계획내용’은 라벨과 내용이 한눈에 구분되게 굵게(볼드) 처리한다.
  - 최소 라벨(“기대효과:”, “계획내용:”)은 반드시 볼드
  - 가능하면 라벨+값 전체를 볼드로 하되, 가독성이 떨어지면 라벨만 볼드로 한다(둘 중 더 보기 좋은 쪽 선택)
- 값이 비어 있지 않도록 생성/보정 로직을 통해 항상 채워진 값이 미리보기에서 보이게 한다.

────────────────────────────────────────
[2] 기대효과 100자 제한 미반영(매우 중요, 이중 안전장치 필수)
[문제]
- 작성지침에 기대효과 100자 제한이 있는데 실제 생성/출력에 반영되지 않는다.

[요구사항]
- 기대효과는 공백 포함 100자 이내를 “항상” 만족해야 한다.
- 적용 위치는 반드시 2곳(이중 안전장치):
  (A) 데이터 생성 직후(샘플/AI/파싱 직후) 100자 보정
  (B) 워드 출력 직전(문서 생성 직전) 100자 보정
- 100자 초과 시 처리:
  - 의미 유지한 채 자동 축약하여 100자 이내로 만든다(단순 잘라내기보다 축약 우선)
  - 축약 후에도 문장이 1문장 이상 남아야 한다(공란 금지)

────────────────────────────────────────
[3] 5대영역(보호/교육/문화/정서지원/지역사회연계) 워드 페이지 나눔(필수)
[문제]
- 5대영역이 페이지로 구분되지 않아 후처리 작업이 불편하다.

[요구사항]
- 워드 출력(docx)에서 5대영역은 “영역 시작마다 새 페이지”로 분리한다.
  - 보호 → 교육 → 문화 → 정서지원 → 지역사회연계
  - 각 영역 제목이 새 페이지 상단에서 시작되도록 page break 삽입
- 표가 영역 경계에서 애매하게 잘리지 않도록(가능한 범위에서) 영역 단위로 배치한다.

────────────────────────────────────────
[4] 평가계획: 기대효과 연동 + 가로폭 비율 확정(캡처 기준 5열)
[현재 평가계획 표 구조(확정)]
- 5열: 세부영역 / 프로그램명 / 기대효과 / 평가계획 / 평가방법

[요구사항 4-1 | 기대효과 연동]
- 평가계획의 ‘기대효과’ 값은 세부사업내용 표의 ‘기대효과’를 “그대로” 재사용한다.
- 별도의 문장 생성/변형 금지(동일 텍스트 원칙).
- 세부사업내용 기대효과가 100자 보정되면 평가계획도 동일하게 보정된 값이 들어가야 한다.

[요구사항 4-2 | 평가계획 표 가로폭]
- 기대효과 컬럼 폭을 가장 넓게 한다.
- 나머지 4개 컬럼은 균등하게 한다.
- 비율(확정):
  - 세부영역 15% / 프로그램명 15% / 기대효과 40% / 평가계획 15% / 평가방법 15%
  => ratios=[0.15, 0.15, 0.40, 0.15, 0.15]
- 워드에서 자동맞춤(autofit) 때문에 폭이 깨지지 않도록 fixed layout + 폭 강제 적용을 한다.

────────────────────────────────────────
[5] Part2 세부사업내용 표 폭 규칙 유지(7열)
- 세부사업내용 표(7열) 비율은 기존 규칙 유지:
  [세부영역, 프로그램명, 기대효과, 대상, 인원, 주기, 계획내용]
  - 8% / 8% / 30% / 8% / 8% / 8% / 30%
  => ratios=[0.08,0.08,0.30,0.08,0.08,0.08,0.30]

────────────────────────────────────────
[구현 지침(필수)]
1) UI(문서형태 미리보기)
- ‘기대효과’가 렌더링에서 누락된 원인을 찾아 반드시 표시되게 수정한다(데이터 경로/컴포넌트/조건분기 점검).
- 기대효과/계획내용 라벨 및 표시를 볼드로 처리한다(미리보기에서 즉시 구분되게).

2) 워드(docx)
- 5대영역 시작마다 page break 삽입
- 세부사업내용 표(7열): ratios=[0.08,0.08,0.30,0.08,0.08,0.08,0.30] 적용
- 평가계획 표(5열): ratios=[0.15,0.15,0.40,0.15,0.15] 적용
- 평가계획 기대효과 = 세부사업내용 기대효과를 그대로 사용

3) 100자 보정 로직
- 생성 직후 + 워드 출력 직전 두 번 적용(이중 안전장치)
- 보정 전/후 길이를 로그로 남겨 검증 가능하게 한다(디버그 로그로 충분)

────────────────────────────────────────
[검증 체크리스트(OK/NG로 제출)]
- [ ] 문서형태 미리보기에서 모든 항목에 기대효과/계획내용이 모두 표시된다.
- [ ] 미리보기에서 ‘기대효과:’, ‘계획내용:’이 볼드로 구분되어 보인다.
- [ ] 기대효과 100자 초과 0건(UI/워드 모두).
- [ ] 워드에서 5대영역이 페이지 나눔으로 분리된다.
- [ ] 평가계획 기대효과가 세부사업내용 기대효과와 동일하다(연동).
- [ ] 평가계획 표 비율이 15/15/40/15/15로 적용되어 기대효과 칸이 가장 넓다.
- [ ] 세부사업내용 표 비율 8/8/30/8/8/8/30이 유지된다.

[자체 테스트 필수]
- 수정 후 반드시 아래 2가지 케이스로 직접 실행 테스트하고, 결과(OK/NG)와 근거(스크린샷/로그)를 남겨줘.
  1) 샘플 데이터 로드 → PART2 미리보기(기대효과/계획내용 표시 + 100자) 확인 → 워드 출력(페이지 나눔 + 표폭) 확인
  2) 실제 업로드(PDF 등) → AI 분석 → PART2 미리보기(기대효과/계획내용 표시 + 100자) 확인 → 워드 출력(페이지 나눔 + 표폭) 확인
- 테스트를 통과하지 못하면 수정 완료로 간주하지 말고, 원인 수정 후 재테스트까지 반복해줘.

[결과 보고 방식]
- 변경한 파일 목록 + 각 파일에서 무엇을 왜 바꿨는지 요약
- 체크리스트 항목별 OK/NG + 근거(스크린샷/로그) 첨부
