목표: “분류는 정상인데, 지침 기반 재작성(특히 표 셀 300자 / 만족도 분석·제언 500자)”이 계속 실패한다. 
UI에서 글자수/불릿 규칙을 통과하도록 ‘표 셀’과 ‘만족도 섹션’에도 min 글자수를 강제 적용하고, 미리보기에서 불필요한 블록(그림1,2)을 제거한다.

[현상 요약]
1) 본문 텍스트 영역(예: 이용아동 욕구, 지역특성, 사업목적/목표)은 글자수 성공.
2) 표(환류계획 등) 안의 셀 텍스트가 “300자 조건”에서 실패.
3) 만족도 문항 분석/종합 제언(각 500자)도 글자수 실패.
4) 미리보기에서 불필요한 안내/검증 화면(그림1,2)은 삭제 필요.

[원인 가설]
- guidelines_template.json(또는 기본 규칙)에서 표 컬럼에는 max만 있고 min이 없다.
- utils.py의 _apply_table_rule()는 bullet_count와 max 자르기만 하고, min 충족을 위한 padding/리라이트를 하지 않는다.
- 만족도 분석/제언은 apply_guidelines_to_analysis() 적용 대상이 아니거나, min 강제가 없다.
- UI 검증은 ‘표 셀도 최소 300자’, ‘만족도도 최소 500자’로 보고 있는데 생성물이 짧아서 실패.

[필수 수정 사항]
A. 규칙(JSON) 보강
- guidelines_template.json의 다음 표 컬럼에 min_chars_no_space를 추가:
  - part1.feedback_table.columns.problem: min_chars_no_space = 200~300 중 요구에 맞게(요구가 “300자”면 300)
  - part1.feedback_table.columns.improvement: min_chars_no_space = 300
  - part2.detail_table.columns.content: min_chars_no_space = 300 (요구가 있으면)
  - part4.budget_table/details 등은 요구 없으면 제외
- 만족도 섹션(주관식 요약분석, 종합 분석 및 제언)에 대해서도 규칙을 신설:
  - satisfaction.subjective_summary: min_chars_no_space=500, max_chars_no_space=700, format="paragraph"
  - satisfaction.overall_suggestion: min_chars_no_space=500, max_chars_no_space=700, format="paragraph"
  (실제 앱 데이터 구조/키에 맞춰 반영)

B. utils.py 후처리 강화(핵심)
1) _apply_table_rule()이 각 셀의 col_rule에서 min_chars_no_space도 읽어서,
   - 공백 제외 글자수가 min 미만이면 자동으로 padding 또는 리라이트로 채우게 수정한다.
2) padding은 ‘(추가 내용 필요)’ 같은 placeholder 금지.
   - 자연스러운 보강 문장 리스트(PADDING_PHRASES)를 만들어 순환 추가.
3) bullet_count가 있는 셀은:
   - 먼저 불릿 개수 맞추기
   - 그 다음 min 글자수 채우기(각 불릿 문장에 자연스럽게 1~2문장 덧붙이기)
   - 마지막에 max 초과 시 자르기
4) 만족도 500자도 동일한 규칙 적용(문단이 짧으면 min까지 보강 문장 추가 후 max로 컷).

C. 생성 프롬프트 강화(표/만족도 전용 2단계 옵션)
- 표 셀/만족도 문단이 min을 못 맞춘 경우에만,
  현재 셀 텍스트 + 입력 요약(snippet)을 넣고 “min~max 범위/불릿 개수/불필요한 문구 금지”를 강하게 준 프롬프트로 재생성한다.
- 재시도는 1~2회로 제한.

D. UI(미리보기) 정리: 그림1,2 삭제
- main.py에서 미리보기에 표시되는 불필요한 블록(예: 규칙 코드 상태, 작성지침 적용 로그 등)이 있다면
  - 기본값 숨김(checkbox로만 보기) 또는 완전 제거.
- 사용자가 원하는 것은 ‘작성 결과 폼’ 중심 화면이므로, 디버그/검증 블록은 사이드바(또는 expander 안)로만 유지한다.

[검증(반드시 통과해야 함)]
1) PDF 3개 업로드 → “AI 분석 시작”
2) PART1~PART4 텍스트/표가 모두 채워짐
3) 표 셀(problem/improvement/content 등)이 공백 제외 최소 300자를 충족(요구대로)
4) 만족도 요약분석/종합 제언이 각각 공백 제외 최소 500자 충족
5) 불릿 규칙이 있는 필드는 ‘• ’로 시작하는 줄 수가 정확히 맞음
6) 미리보기에서 그림1,2에 해당하는 블록은 보이지 않음(또는 기본 숨김)

[주의]
- 공백 제외 글자수 계산은 기존 count_chars_no_space() 함수를 그대로 사용.
- placeholder 문구(“추가 내용 필요”)는 최종 결과에 절대 남기지 말 것.
- 수정 파일 후보: guidelines_template.json, utils.py, main.py (필요 시 doc_utils.py)
