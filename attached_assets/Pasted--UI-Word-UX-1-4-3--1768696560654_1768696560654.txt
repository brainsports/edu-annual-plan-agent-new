상·하반기 월간계획 UI + Word 출력 + 개별 다운로드 UX 구현
1. 작업 목적 (최상위 규칙)

참참 4.3 연간사업계획서 생성도우미에서
상반기·하반기 월간계획을 표 기반 UI로 작성하고,
각 반기를 ‘개별 Word 파일’로 다운로드할 수 있도록 구현한다.

⚠️ 전체 다운로드(통합 Word)는 절대 만들지 않는다.

2. 다운로드 구조 절대 규칙 (변경 금지)

사용자는 아래 4개 문서를 각각 따로 다운로드해야 한다.

연간사업계획서 PART 1

연간사업계획서 PART 2

상반기 월간계획

하반기 월간계획

👉 총 4회 다운로드 구조가 반드시 유지되어야 한다.
👉 “전체 다운로드 / 모두 합쳐서 다운로드” 기능은 절대 구현하지 않는다.

입력 즉시 오른쪽 미리보기 반영

6. Word(.docx) 출력 규칙 (월간계획)
6-1. 파일 분리 원칙

상반기 월간계획 → 상반기 전용 Word 파일 1개

하반기 월간계획 → 하반기 전용 Word 파일 1개

❌ 상·하반기를 합친 Word 파일 금지
❌ 연간 전체 묶음 Word 금지

8. 다운로드 UX (DownloadPage)
8-1. 체크리스트 UI
☑ 연간사업계획서 PART 1
☑ 연간사업계획서 PART 2
☑ 상반기 월간계획
☑ 하반기 월간계획


※ 체크는 “선택 표시” 용도이며
※ 묶음 다운로드 의미가 아님

8-2. 다운로드 버튼 규칙

항목별 개별 버튼 제공

예:

[PART 1 Word 다운로드]

[PART 2 Word 다운로드]

[상반기 Word 다운로드]

[하반기 Word 다운로드]

👉 버튼 1회 클릭 = Word 파일 1개
👉 사용자가 4개 모두 누르면 4번 다운로드

9. UX 원칙 (중요)

일부만 작성해도 해당 항목은 다운로드 가능

“모두 작성해야 다운로드 가능” 같은 제한 금지

실무자가 필요한 문서만 골라 제출할 수 있어야 한다

10. 최종 완료 기준 (성공 조건)

상·하반기 월간계획이 월별 표 UI로 완성되어 있다

상·하반기 Word 파일이 각각 따로 다운로드된다

전체 다운로드 기능은 존재하지 않는다

Word 문서는 제출 가능한 품질이다

PART 1 / PART 2 / 월간계획 UX가 하나의 흐름으로 자연스럽다

🔒 최종 고정 문장 (AI가 반드시 기억해야 할 규칙)

참참 4.3은
‘4개의 문서, 4번의 다운로드’ 구조이며
전체 다운로드는 절대 만들지 않는다.
