[상황/문제]
Streamlit 앱에서 Gemini 분석 결과를 JSON으로 파싱하는 과정에서 다음 문제가 반복된다:
- JSON 파싱 오류(Expecting value)
- 코드블록(```json) / 설명문이 섞여 파싱 실패
- 최상위가 list로 반환되어 setdefault 오류 발생
- 표(테이블) 구조가 매번 흔들려 컬럼 누락/형식 불일치 발생

[목표]
향후 “표 기반 작업”이 많아져도 동일하게 적용되도록,
utils.py에 “공통 처리 레이어(표준 파서 + 재요청 + 표 정규화)”를 구현한다.
main.py와 doc_utils.py는 수정하지 않는다.

[수정 범위]
- main.py 수정 금지
- doc_utils.py 수정 금지
- utils.py만 수정
- get_gemini_analysis()는 안정적으로 dict를 반환(실패 시 None)하도록 유지

[필수 구현 사항 - utils.py]
1) 공통 JSON 전처리/추출/파서 유틸 추가
- 코드블록 제거( ```json / ``` 가 문자열 어디에 있든 제거)
- JSON 덩어리만 정확히 추출:
  - 괄호 균형 기반으로 { ... } 또는 [ ... ] 구간만 잘라내기
  - 문자열(따옴표) 내부는 무시하고 depth 카운팅
- json.loads 우선 시도, 실패 시 ast.literal_eval(작은따옴표/파이썬 dict 형태) 시도

2) 최상위 타입 방어
- 파싱 결과가 dict면 그대로 사용
- list면:
  - [ {..} ] 형태(길이1, 내부 dict)면 dict로 펼쳐 사용
  - 그 외 list면 “JSON 객체({})로 다시 출력” 재요청 트리거

3) 자동 재요청(Repair) 로직 추가
- 1차 파싱 실패 또는 최상위가 dict가 아니면,
  “설명문/마크다운/코드블록 금지, 반드시 JSON 객체({}) 1개만 출력, 배열로 시작 금지”
  조건으로 Gemini에 최대 2회 재요청하여 복구
- 재요청에도 실패하면 st.error로 안내하고 None 반환

4) 기본 루트 키 강제
- 반환 dict에는 항상 아래 키가 존재하도록 setdefault로 보정:
  part1_general, part2_programs, part3_monthly_plan, part4_budget_evaluation
- part4_budget_evaluation 기본값: {"budget_table": [], "feedback_summary": []}

5) 표(테이블) 정규화 함수 추가
- normalize_table_rows(rows, columns, fill_value=""):
  - rows가 list가 아니면 []
  - 각 row가 dict가 아니면 스킵
  - columns 기준으로 누락 컬럼은 fill_value로 채워 dict 형태로 통일
- get_gemini_analysis() 반환 직전에 최소 아래 표를 정규화:
  part1_general.feedback_table -> ["area","problem","improvement"]
  part1_general.total_review_table -> ["category","content"]

6) 디버그 로그(필수, 과도하지 않게)
- 콘솔에 아래 2개 출력:
  [Gemini] model=...
  [Gemini raw preview] 응답 앞 500자 (없으면 (EMPTY))
- Repair 수행 시:
  [Gemini repair#1 preview] ...
  [Gemini repair#2 preview] ...

7) 모델/키 처리
- GEMINI_API_KEY 없으면 st.error 안내 후 None 반환
- 모델은 환경변수 GEMINI_MODEL 기본값 "gemini-2.0-flash"
- google.generativeai(genai) SDK 방식 유지
- 가능하면 generate_content에 response_mime_type="application/json" 적용(지원 안되면 예외 없이 기존 호출)

[완료 조건]
- “AI 분석 시작” 클릭 시 앱이 크래시하지 않는다.
- Gemini 응답이 코드블록/설명문을 섞어도 대부분 정상 파싱된다.
- 표 컬럼 누락/형식 흔들림이 normalize_table_rows로 고정된다.
- 수정은 utils.py에만 반영된다.
