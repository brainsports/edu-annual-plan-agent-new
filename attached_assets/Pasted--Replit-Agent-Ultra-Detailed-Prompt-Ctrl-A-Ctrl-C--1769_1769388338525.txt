### 💡 이 프롬프트가 '초정밀'인 이유 (차별점)

1.  **파일의 분리 (`Project File Structure`)**:
    * 이전에는 `main.py` 하나에 다 때려 넣었지만, 이번에는 `utils.py`(AI 통신), `doc_utils.py`(워드 생성)로 **기능을 쪼개달라고 명시**했습니다. 이렇게 해야 코드가 길어져도 에러 찾기가 쉽고, AI가 코드를 짤 때도 헷갈리지 않습니다.

2.  **Dockerfile의 한글 설정 강제 (`fonts-nanum`)**:
    * 가장 중요한 부분입니다. 클라우드 런에 배포했을 때 그래프의 한글이 'ㅁㅁㅁ'으로 깨지는 것을 막기 위해 `apt-get install fonts-nanum` 명령어를 **Docker 설정에 콕 집어 넣었습니다.**

3.  **JSON 스키마의 구체화**:
    * AI에게 "알아서 JSON 줘"가 아니라, `part1`, `part2_programs` 등 **키(Key) 이름과 데이터 타입(String, Int)까지 지정**했습니다. 이렇게 하면 Streamlit 화면과 데이터가 어긋날 일이 없습니다.

4.  **세션 상태(Session State) 관리 명시**:
    * Streamlit의 고질적인 문제인 "버튼 누르면 데이터 날아가는 현상"을 막기 위해 `st.session_state`를 반드시 사용하라고 지시했습니다.

### 🏃‍♂️ 실행 방법

1.  **Replit**에서 새 Python 프로젝트를 엽니다.
2.  하단이나 우측의 **Agent(AI)** 창을 엽니다.
3.  위의 영어로 된 프롬프트를 그대로 붙여넣습니다.
4.  Agent가 파일을 다 만들고 나면, **[Run]** 버튼을 눌러 테스트합니다.
5.  잘 작동하면 우측 상단의 **[Deploy]** -> **[Google Cloud Run]**을 선택하거나, 깃허브를 통해 배포합니다.

이 프롬프트라면 레플릿 에이전트가 "이 사람은 고수다"라고 인식하고, 대충 짜는 코드가 아니라 실무 수준의 코드를 작성해 줄 것입니다.
